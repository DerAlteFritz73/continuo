# Basso Continuo Realizer

Automatic realization of unfigured bass lines with intelligent key detection and harmonic analysis.

## How it works

### 1. File Import & Parsing

When you import a MusicXML file (.xml, .musicxml, or .mxl):
- The parser extracts the bass line (lowest-pitched notes)
- Identifies any existing figures or figured bass notation
- Extracts the melody/soprano voice if present (especially useful for keyboard parts)
- Reads the key signature and time signature from the file

### 2. Passage Detection & Key Inference

The system automatically divides the piece into **passages** and detects the correct key for each:

#### How Passages Are Detected
Passages are divided at:
1. **Explicit key signature changes** in the MusicXML file (highest priority)
2. **Cadential points** - measures with long notes (half notes, dotted notes) that indicate harmonic resolution
3. **Default intervals** - if no clear cadences, passages are divided every 4-8 measures

#### How Keys Are Detected
For each passage, the system analyzes the bass line using multiple signals:

| Signal | How it works | Strength |
|--------|-------------|----------|
| **Final Note** | The last non-rest bass note in the passage is usually the tonic (home key) | Strongest |
| **Accidental Pattern** | Raised 7ths (leading tones) indicate the tonality. E.g., many G# notes suggest D major | Strong |
| **Cadential Motion** | A 4th or 5th leap in the bass (like V→I motion) strongly indicates the final key | Strong |
| **Pitch Histogram** | Distribution of note frequencies; the most common note often indicates the key center | Weak-Moderate |

**Example:** In a piece with key signature C major but ending on F:
- Final note: F → suggests F major or F minor
- Cadence: Bb→F motion → confirms V→I in F
- Accidentals: Bb appears often → confirms F major (Bb is the 4th)
- Result: **F major detected** (even though file says C major)

#### Confidence Levels
- **High**: 3+ signals agree (final note + accidental pattern + cadence all point to same key)
- **Medium**: 2 signals agree
- **Low**: Only 1 signal present

### 3. Figure Inference from Melody

If the file includes an upper voice (melody/soprano) but no figured bass notation:

1. **Extract melody notes** from the staff 1 (treble) voice
2. **Analyze each melody note** against the bass note at the same time
3. **Infer the interval** from bass to soprano (is it a 3rd? 5th? 6th?)
4. **Determine the chord type**:
   - Soprano on 3rd or 5th → root position (figure 5)
   - Soprano on 6th → first inversion (figure 6)
   - Soprano on 4th → second inversion (figure 4/3)
5. **Source marked as "melody"** in the output

**Example:** 
- Bass: C, Soprano: E → interval of 3rd → infers figure 5 (C major root position)
- Bass: G, Soprano: B → interval of 3rd above G → infers figure 5 (G major root position)

### 4. Chord Realization

For each bass note, the system:
1. Determines the **scale degree** (1st, 2nd, 3rd, etc. of the key)
2. Uses the **figured bass** (from file or inferred) to select intervals
3. Applies **voice-leading rules** to place upper voices (soprano, alto, tenor):
   - Minimize voice movement from the previous chord
   - Avoid parallel 5ths and octaves
   - Resolve leading tones and chordal 7ths where required
   - Constrain soprano to match melody notes if provided

### 5. Output

The system produces:
- **Realized MusicXML** with all 4 voices written out
- **Passages panel** showing detected keys for each section
- **Figure sources** labeled as:
  - `file` - figures provided in the input
  - `melody` - figures inferred from soprano voice
  - `computed` - figures generated by decision tree

## Why This Matters

### Problem It Solves

**Scenario:** You export a MuseScore file with:
- A hand-written bass line
- A pre-composed melody
- But the wrong key signature (e.g., C major instead of F major)

**Before:** The system would misinterpret chromatic notes, producing wrong harmonies.

**After:** 
- Passage detection identifies F major from the bass motion
- Figure inference uses your melody to confirm the harmony
- Upper voices are added correctly in F major, not C major

### Real-World Use Case

Importing historical continuo data (bass + melody from IMSLP) where:
1. Key signatures may be ambiguous or wrong
2. No figured bass notation exists
3. You want automatic 4-voice realization that respects the melody

## Technical Implementation

- **KeyDetector** service: Analyzes bass notes to infer keys
- **PassageDetector** service: Divides scores and calls KeyDetector per passage
- **FigureInference** service: Determines figures from melody notes
- **VoiceLeadingEngine**: Places upper voices respecting detected keys and inferred figures

See `/docs/` for detailed implementation notes.
